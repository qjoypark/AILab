### 2. 测试覆盖范围

#### 2.1 审批流程流转逻辑测试 (9个测试用例)

1. **单级审批流程 - 审批通过后状态正确更新**
   - 验证审批通过后申请单状态从"审批中"变为"审批通过"
   - 验证当前审批人字段被清空
   - 验证审批状态正确更新

2. **单级审批流程 - 审批拒绝后状态正确更新**
   - 验证审批拒绝后申请单状态变为"审批拒绝"
   - 验证审批流程终止
   - 验证当前审批人字段被清空

3. **多级审批流程 - 第一级通过后流转到第二级**
   - 验证第一级审批通过后流程继续
   - 验证当前审批人更新为第二级审批人
   - 验证申请单仍处于"审批中"状态
   - 验证审批记录正确记录

4. **多级审批流程 - 中间级拒绝后流程终止**
   - 验证第一级通过、第二级拒绝的场景
   - 验证流程在中间级拒绝后终止
   - 验证最终状态为"审批拒绝"
   - 验证两级审批记录都被正确记录

5. **审批记录完整性 - 所有必需字段都被记录**
   - 验证审批记录包含所有必需字段：
     - 申请单ID和申请单号
     - 审批人ID和姓名
     - 审批层级
     - 审批结果
     - 审批意见
     - 审批时间
     - 创建时间

#### 2.2 库存检查测试 (5个测试用例)

1. **库存检查 - 库存充足时允许创建申请**
   - Mock库存充足的场景
   - 验证申请创建成功

2. **库存检查 - 库存不足时拒绝创建申请**
   - Mock库存不足的场景
   - 验证抛出BusinessException
   - 验证错误消息包含库存不足信息和当前可用库存

3. **库存检查 - 多个物料时检查所有物料库存**
   - 创建包含多个物料的申请
   - Mock第一个物料库存充足，第二个物料库存不足
   - 验证整个申请被拒绝

4. **库存检查 - 申请数量为零时拒绝**
   - 验证申请数量为0时抛出异常
   - 验证错误消息为"申请数量必须大于0"

5. **库存检查 - 申请数量为负数时拒绝**
   - 验证申请数量为负数时抛出异常
   - 验证错误消息为"申请数量必须大于0"

#### 2.3 权限控制测试 (6个测试用例)

1. **权限控制 - 只有当前审批人可以审批**
   - 使用错误的审批人ID尝试审批
   - 验证抛出BusinessException
   - 验证错误消息包含"不是当前审批人"

2. **权限控制 - 只有申请人可以取消申请**
   - 使用其他用户ID尝试取消申请
   - 验证抛出BusinessException
   - 验证错误消息为"只有申请人可以取消申请"

3. **权限控制 - 申请人可以成功取消自己的申请**
   - 验证申请人可以取消自己的申请
   - 验证申请状态变为"已取消"

4. **权限控制 - 已审批通过的申请不能取消**
   - 创建并审批通过申请
   - 尝试取消已审批通过的申请
   - 验证抛出BusinessException
   - 验证错误消息为"当前状态不允许取消"

5. **权限控制 - 已出库的申请不能取消**
   - 创建申请并手动设置状态为"已出库"
   - 尝试取消
   - 验证抛出BusinessException
   - 验证错误消息为"当前状态不允许取消"

6. **权限控制 - 不在审批中的申请不能审批**
   - 创建申请并手动设置状态为"已完成"
   - 尝试审批
   - 验证抛出BusinessException
   - 验证错误消息包含"不在审批中状态"

### 4. 测试设计特点

#### 4.1 使用Spring Boot Test框架
- 使用 `@SpringBootTest` 注解进行集成测试
- 使用 `@Transactional` 确保测试数据隔离
- 使用 `@MockBean` Mock外部依赖（InventoryClient, MaterialClient）

#### 4.2 测试数据准备
- 在 `@BeforeEach` 方法中准备测试数据
- 清理旧数据，确保测试环境干净
- 设置审批流程配置（普通领用单级审批、危化品多级审批）
- Mock材料信息和库存检查

#### 4.3 辅助方法
提供了多个辅助方法简化测试代码：
- `createNormalApplicationRequest()`: 创建普通领用申请
- `createHazardousApplicationRequest()`: 创建危化品领用申请
- `getApprovalRecords()`: 获取审批记录
- `setupFlowConfigs()`: 设置流程配置
- `setupMaterialMocks()`: 设置材料Mock

## 测试质量保证

### 1. 测试隔离
- 每个测试方法使用 `@Transactional` 注解，测试后自动回滚
- `@BeforeEach` 方法清理测试数据，确保测试独立性

### 2. Mock策略
- 外部服务（InventoryClient, MaterialClient）使用Mock
- 数据库操作使用真实的H2数据库
- 确保测试的可靠性和可重复性

### 3. 断言完整性
- 使用JUnit 5的断言方法
- 验证状态、数据、异常消息
- 确保测试覆盖所有关键路径

### 4. 可读性
- 使用 `@DisplayName` 注解提供清晰的测试描述
- 测试方法命名遵循 `test{场景}_{预期结果}` 模式
- 代码注释说明测试步骤
