## 关键设计决策

### 1. 数量验证策略
- **决策**: 严格验证 `领用数量 = 实际使用量 + 归还量 + 废弃量`
- **理由**: 确保账实相符，防止数据不一致
- **影响**: 用户必须准确填写三个数量字段

### 2. 库存服务容错
- **决策**: 库存服务调用失败时记录日志但不阻止使用记录更新
- **理由**: 使用记录的准确性比库存更新更重要，库存可通过盘点调整
- **影响**: 需要定期核对库存与使用记录

### 3. 状态管理
- **决策**: 归还后状态从"使用中"(1)变为"已归还"(2)
- **理由**: 明确区分使用中和已归还的记录
- **影响**: 防止重复归还，支持后续审计

### 4. 备注追加
- **决策**: 保留原有备注并追加新备注
- **理由**: 保持完整的操作历史
- **影响**: 备注字段可能变长，需要足够的存储空间

### 5. 事务管理
- **决策**: 使用 `@Transactional` 确保数据一致性
- **理由**: 防止部分更新导致数据不一致
- **影响**: 提高数据可靠性，但可能影响并发性能
